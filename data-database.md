## Data Base Concepts

- **Database** are a data storage system that organizes and store data into structured tables. many use SQL and security is a critical requirement. When crreating a database you must input data either manually or by importing it for other sources like spreadshhets, forms and other databases.
- **Quesries** is a request to get data from a database. Normally done in SQL.
- **Reports** are made when data from an Query is used and changed into something easy to read such as summaries, graphs and tables.

## Database Verses Spreadsheets (Flat Files)

- **Spreadsheets or flat files** are programs for small to medium sets of data (normally numbers) and for a limited number of users.
- **Databases** offer faster processing, better back up options, controlled data types, support many users at once and have better security.

### Databases Can be:
- Offline - better against cyber attacks and does not require internet.
- Online - Supports many users at once, requires internet.
- Local - Fast data access and better security.
- Cloud - data available amlost all the time and scable.

## Database Structure

There are three main types odf database structures.
- **Structured data** - Data divided into clear, seperate categories. Ideal for relational databases
- **Non-structured data** - Data without a clear pattern. Uses XML or HTML to keep it organized with the use of "tags"
- **Semi-structured data** - Used for data that does not fall into neat rows or columns but are simular. great for **key-value** databases.

## Relational Databases

- **Schema** is the architecture of how a database is structured and organized.
- **Entity** i.e name (product or person)
- **Attruibute** is data of the entity
- **Relastionship** is the connectiuon between the entity and attribute

Columns are call **fields** which is one detail about each record in a database.
Rows are call **records** and are all the fields together.

- **Primary Key** - unique identifier for a record so duplicates cannot happen. Used as a reforence to a record.
- **Foregn Key** - a field that is a primarty key in another table.

### Constraints

Databases also have constraints such as a **not null** which mean a field cannot be left blank i.e. requires input.
